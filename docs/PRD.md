# Claude Launcher 产品需求文档 (PRD)

> 版本: 1.0
> 日期: 2026-03-28
> 状态: 草案

---

## 1. 项目概述

Claude Launcher 是一款轻量级 Windows 桌面应用程序，帮助用户管理和启动跨多个项目文件夹的 Claude Code CLI 会话。

### 1.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 框架 | Tauri 2.x |
| 后端 | Rust |
| 前端 | Solid.js |
| 运行平台 | Windows 10 (1803+) / Windows 11 |

### 1.2 前置条件

- 用户已安装 Claude Code CLI 并将其加入系统 PATH
- 用户拥有 `~/.claude/projects/` 目录下的会话数据

---

## 2. 核心功能

### 2.1 文件夹管理

用户可配置需要显示的项目文件夹列表。侧边栏展示所有已配置的文件夹，每个文件夹旁显示会话数量角标。

**详细说明:**
- 支持添加、移除、重新排序项目文件夹
- 文件夹路径验证（检查路径是否存在）
- 侧边栏实时显示每个文件夹关联的会话数量
- 配置数据持久化存储到本地（如 `%APPDATA%/claude-launcher/config.json`）

### 2.2 会话浏览器

选中某个文件夹后，列出该文件夹下所有历史 Claude Code 会话。

**详细说明:**
- 会话数据来源: `~/.claude/projects/<encoded-path>/*.jsonl`
- 每条会话显示以下信息:
  - 会话日期（基于文件修改时间或会话内记录的时间戳）
  - 使用的模型（如 claude-sonnet-4-20250514、claude-opus-4-20250514 等）
  - 摘要（首条用户消息的内容，截断显示）
  - 消息数量
- 默认按时间倒序排列（最新优先）
- 支持搜索/过滤会话

### 2.3 会话操作

用户可恢复已有会话或创建新会话。

**详细说明:**
- **恢复会话**: 执行 `claude --resume <session-id>`，在对应文件夹的工作目录下启动
- **新建会话**: 执行 `claude`，在对应文件夹的工作目录下启动
- 所有会话均在新的终端窗口中启动，与 Launcher 进程分离

### 2.4 DNS 健康检查

应用启动时及每隔 5 分钟自动解析 `claude.ai` 的 IP 地址，检测 DNS 是否被污染。

**详细说明:**
- 预期正确 IP: `198.3.16.159`
- 解析结果匹配时，显示绿色状态指示
- 解析结果不匹配时，显示红色警告横幅，提示用户 DNS 可能被污染
- 检查间隔: 应用启动时立即执行一次，之后每 5 分钟执行一次
- 警告状态下仍允许用户启动会话，但需二次确认

### 2.5 启动 Claude

在新终端窗口中生成 Claude Code 进程，与 Launcher 完全分离。

**详细说明:**
- 使用 `cmd /c start` 方式启动新终端窗口
- 启动后 Launcher 进程不追踪子进程状态
- 支持自定义启动参数（预留扩展）

## 3. 用户故事

### US-01: 添加项目文件夹

**作为** Claude Code 用户，
**我希望** 将常用的项目文件夹添加到 Launcher 中，
**以便** 我能快速访问各个项目的 Claude 会话。

**验收标准:**
- 点击"添加文件夹"按钮可打开系统文件夹选择对话框
- 选择后文件夹出现在侧边栏列表中
- 不可重复添加同一文件夹
- 添加无效路径时显示错误提示
- 重启应用后已添加的文件夹仍然存在

### US-02: 查看文件夹会话列表

**作为** Claude Code 用户，
**我希望** 点击侧边栏中的某个项目文件夹后看到该项目的所有历史会话，
**以便** 我能快速了解该项目的历史交互记录。

**验收标准:**
- 点击文件夹后右侧面板展示会话列表
- 每条会话显示日期、模型、摘要（首条用户消息）和消息数量
- 列表按时间倒序排列
- 100 条会话以内加载时间不超过 500ms
- 无会话时显示空状态提示

### US-03: 恢复历史会话

**作为** Claude Code 用户，
**我希望** 能够恢复之前的某个 Claude 会话，
**以便** 我能继续未完成的对话而不丢失上下文。

**验收标准:**
- 点击会话条目上的"恢复"按钮
- 弹出新的终端窗口并执行 `claude --resume <session-id>`
- 终端工作目录为对应的项目文件夹
- 终端窗口独立于 Launcher 运行
- Launcher 在启动后保持可用状态

### US-04: 创建新会话

**作为** Claude Code 用户，
**我希望** 能为某个项目文件夹快速创建新的 Claude 会话，
**以便** 我能开始新的编码任务。

**验收标准:**
- 选中文件夹后可点击"新建会话"按钮
- 弹出新的终端窗口并在对应目录下执行 `claude`
- 终端窗口独立于 Launcher 运行
- 新会话在下次刷新后出现在会话列表中

### US-05: 查看 DNS 健康状态

**作为** Claude Code 用户，
**我希望** 在应用中实时看到 DNS 解析状态，
**以便** 我能知道当前网络环境是否安全可用。

**验收标准:**
- 应用启动后立即执行一次 DNS 检查
- DNS 正常时状态栏显示绿色标识和"DNS 正常"文字
- DNS 异常时显示红色警告横幅，包含解析到的错误 IP 地址
- 每 5 分钟自动执行一次检查
- 支持手动点击刷新立即重新检查

### US-06: DNS 异常时启动会话的风险提示

**作为** Claude Code 用户，
**我希望** 在 DNS 异常时启动会话能收到明确的风险警告，
**以便** 我能在了解风险的情况下做出决策。

**验收标准:**
- DNS 异常状态下点击"恢复"或"新建会话"按钮
- 弹出确认对话框，内容包含:"DNS 解析异常，当前解析 IP 为 X.X.X.X，预期为 198.3.16.159，存在 DNS 污染风险，是否继续?"
- 用户可选择"继续启动"或"取消"
- 用户选择"继续启动"后正常执行会话启动流程

### US-07: 移除项目文件夹

**作为** Claude Code 用户，
**我希望** 能从 Launcher 中移除不再需要的项目文件夹，
**以便** 保持侧边栏简洁。

**验收标准:**
- 右键文件夹或点击删除按钮可触发移除操作
- 移除前显示确认提示
- 移除后文件夹从侧边栏消失
- 仅移除 Launcher 中的配置引用，不删除实际文件夹或会话数据
- 重启应用后已移除的文件夹不再显示

### US-08: 查看会话数量角标

**作为** Claude Code 用户，
**我希望** 在侧边栏的每个文件夹旁看到该文件夹的会话数量，
**以便** 我能快速了解各项目的活跃程度。

**验收标准:**
- 每个文件夹名称旁显示会话数量角标（如 `My Project (12)`）
- 数量基于 `~/.claude/projects/<encoded-path>/` 下的 `.jsonl` 文件数
- 添加新文件夹后角标立即显示
- 文件夹会话为 0 时显示 `(0)` 而非隐藏角标

### US-09: 快速搜索会话

**作为** Claude Code 用户，
**我希望** 能在会话列表中搜索关键词，
**以便** 快速找到某个特定的历史会话。

**验收标准:**
- 会话列表上方提供搜索输入框
- 输入关键词后实时过滤匹配的会话（匹配摘要内容）
- 搜索结果保持时间倒序排列
- 清空搜索框后恢复完整列表
- 搜索无结果时显示"无匹配会话"提示

### US-10: 应用快速启动

**作为** Claude Code 用户，
**我希望** 应用能在 1 秒内完成启动并显示主界面，
**以便** 我不必等待即可开始工作。

**验收标准:**
- 从双击图标到主界面完全渲染不超过 1 秒
- 侧边栏文件夹列表立即加载（来自本地缓存配置）
- DNS 检查和会话数量统计在后台异步完成，不阻塞界面渲染
- 加载中的数据使用骨架屏或 loading 状态占位

## 4. 非功能性需求

### 4.1 性能要求

| 指标 | 目标值 | 备注 |
|------|--------|------|
| 应用启动时间 | < 1 秒 | 双击到主界面完全可交互 |
| 内存占用 | < 50 MB | 正常使用状态下的常驻内存 |
| 安装包体积 | < 10 MB | NSIS/MSI 安装包 |
| 会话列表加载 | < 500 ms | 100 条会话以内 |
| DNS 检查耗时 | < 2 秒 | 单次 DNS 解析完成时间 |

### 4.2 兼容性

- **操作系统**: Windows 10 (版本 1803 及以上) / Windows 11
- **架构**: x86_64（优先），可选 ARM64 支持
- **依赖**: Claude Code CLI 已安装并加入 PATH
- **WebView**: 使用系统自带 WebView2（Edge Chromium 内核）

### 4.3 安全性

- 不存储、不传输用户的 Claude API 密钥
- 仅读取本地 `.jsonl` 会话文件，不做任何修改
- DNS 检查仅解析域名，不发送额外网络请求
- 配置文件使用用户级别存储路径（`%APPDATA%`），遵循最小权限原则

### 4.4 可用性

- 界面语言: 简体中文
- 支持系统深色/浅色模式自适应
- 侧边栏文件夹支持拖拽排序
- 主要操作支持键盘快捷键

### 4.5 可维护性

- 前后端代码分离，职责清晰
- Rust 后端负责: 文件系统操作、DNS 解析、进程管理
- Solid.js 前端负责: 界面渲染、用户交互、状态管理
- 通过 Tauri IPC 通信，接口定义明确

## 5. 技术架构概览

```
+--------------------------------------------------+
|                Claude Launcher                    |
|                                                   |
|  +-------------------+   +---------------------+ |
|  |   Solid.js 前端    |   |    Rust 后端        | |
|  |                   |   |                     | |
|  | - 侧边栏文件夹列表 | <-> | - 读取配置文件      | |
|  | - 会话列表浏览     |IPC | - 扫描 .jsonl 会话  | |
|  | - DNS 状态显示     | <-> | - DNS 解析检查      | |
|  | - 启动操作触发     |   | - 进程启动管理      | |
|  +-------------------+   +---------------------+ |
|                                                   |
+--------------------------------------------------+
        |                          |
        v                          v
  系统 WebView2              cmd /c start claude
  (Edge Chromium)            (独立终端窗口)
```

### 5.1 Tauri IPC 命令设计

| 命令 | 方向 | 说明 |
|------|------|------|
| `get_folders` | 前端 -> 后端 | 获取已配置的文件夹列表 |
| `add_folder` | 前端 -> 后端 | 添加项目文件夹 |
| `remove_folder` | 前端 -> 后端 | 移除项目文件夹 |
| `reorder_folders` | 前端 -> 后端 | 更新文件夹排序 |
| `get_sessions` | 前端 -> 后端 | 获取指定文件夹的会话列表 |
| `resume_session` | 前端 -> 后端 | 恢复指定会话 |
| `new_session` | 前端 -> 后端 | 创建新会话 |
| `check_dns` | 前端 -> 后端 | 执行 DNS 健康检查 |
| `dns_status_changed` | 后端 -> 前端 | DNS 状态变更事件 |

### 5.2 数据模型

```
Folder {
  path: String        // 项目文件夹绝对路径
  name: String        // 显示名称（文件夹名）
  session_count: u32  // 会话数量
}

Session {
  id: String           // 会话 ID（文件名去除 .jsonl 后缀）
  folder_path: String  // 所属项目文件夹路径
  date: DateTime       // 会话日期
  model: String        // 使用的模型
  summary: String      // 首条用户消息摘要
  message_count: u32   // 消息数量
}

DnsStatus {
  healthy: bool        // 是否正常
  resolved_ip: String  // 实际解析到的 IP
  expected_ip: String  // 预期 IP (198.3.16.159)
  checked_at: DateTime // 最近检查时间
}

AppConfig {
  folders: Vec<Folder>  // 已配置的文件夹列表
  theme: String         // 主题偏好 (auto/light/dark)
}
```

## 7. 里程碑计划

| 阶段 | 内容 | 预估周期 |
|------|------|----------|
| M1: 基础框架 | Tauri 项目搭建、基本窗口、IPC 骨架 | 2 天 |

| M4: 会话操作 | 恢复/新建会话、终端启动 | 2 天 |

## 8. 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| `.jsonl` 文件格式变更 | 会话解析失败 | 解析器容错处理，跳过无法解析的条目 |
| Claude Code CLI 未安装 | 无法启动会话 | 启动时检测 CLI 可用性，不可用时提示安装指引 |
| 大量会话文件导致加载缓慢 | 用户体验下降 | 实现分页加载、延迟加载，仅解析必要字段 |
| WebView2 未安装 | 应用无法启动 | 安装包内嵌 WebView2 引导安装程序 |
| DNS 检查误报 | 用户困惑 | 支持用户自定义预期 IP，提供"忽略"选项 |
